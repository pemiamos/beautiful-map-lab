# 最美地图研究所

> 一个基于真实教学现场的地图信息设计批评数据库。
>
> 学生留下成长轨迹，默寺留下批评方法，AI 留下辅助痕迹。

这不是一个项目管理仓库，不是档案柜，是一套**持续显影教学思想的系统**。

---

## 这是什么

「最美地图研究所」是默寺老师指导的学生信息可视化项目组。研究主题集中在：

- 中国地理与自然志的信息可视化（湖泊、山地、河流、城市变迁）
- 高密度信息图的版面设计与叙事结构
- AI 辅助下的设计教学方法论

本仓库记录每周例会、点评、设计决策和方法沉淀，长期积累为：

1. **项目管理**：知道每位同学做到哪一步
2. **教学诊断**：知道每位同学卡在哪里
3. **方法沉淀**：知道哪些批评可以转化为设计原则
4. **研究资产**：AI 时代设计教育的真实案例库

## 三条铁律（仓库宪法）

### 铁律一：AI 整理判断，AI 不产生判断

默寺已经做出了所有教学判断。AI 的工作是把口语点评整理成档案文本，**不是分析后给出建议供默寺采纳**。

所有 AI 段落必须用引用块和 `[AI 建议]` / `[AI 补充]` 标注，与默寺整理稿视觉隔离。

### 铁律二：保留默寺的语气指纹

整理稿必须保留默寺的判断词汇（踢给 AI、护城河、抓瞎、大而化之）、转折逻辑、点名节奏。

档案五年后翻看，要能听见默寺当年说话的声音。

### 铁律三：methods/ 必须人工审核才能进

`methods/method-candidates/` 是 AI 候选区。任何条目进入正式 `methods/`，必须由默寺手改一遍。

否则一年后这里全是 AI 味的方法垃圾场。

## 化名机制

学生在仓库里以**自选化名**出现，真名只存在于本地私密文件 `name-mapping.md`（不入库）。

化名不是为了脱敏（仓库公开但不涉及隐私敏感信息），而是为了：

- 让档案语气从"教学"转为"研究"
- 给学生主动权（化名由学生本人选定）
- 降低公开档案对个人的人格化指向

如果你是同行偶然路过本仓库：化名背后是一群正在认真学习信息设计的本科生。请用研究案例的眼光看待他们的成长，而不是把他们当作"被批评的对象"。

## 工作流程

### 每周一次完整循环

```
1. 学生在 Notion 提交周报
       ↓
2. 默寺线上/线下例会，针对每位学生周报口头点评
       ↓
3. 例会录音 + 微信群发言 → raw.md
   （化名替换在这一步完成）
       ↓
4. 三个 AI（Claude / GPT / Gemini）平行调用 cleanup.md
       ↓
5. Claude 用 merge.md 合并三份输出 → notes.md
       ↓
6. 默寺审阅 notes.md（检查语气保留、AI 段落是否越界）
       ↓
7. notes.md 中各学生段落 → 各自的 students/<slug>-2026-Wxx.md
       ↓
8. method-candidates 段落 → methods/method-candidates/
       ↓
9. push 到 GitHub
```

整个流程每周约 30 分钟。

## 标签词典

**所有学生周档案和会议笔记的标签必须从此处取。**

新增标签需先入词典，再在档案中使用。

### AI 与方法论
- `AI幻觉陷阱` — AI 在具体地理转译上的失败
- `AI辅助过度` — 学生把判断踢给 AI
- `肌理缺失` — 设计中缺少手工肌理与材质设计
- `地理转译失败` — AI/学生未能正确把地理信息转为视觉
- `方法论缺失` — 学生缺少可复用的设计原则

### 工程与执行
- `GitHub工程化` — 仓库管理、自动化脚本、版本控制相关
- `执行迟滞` — 任务推进缓慢，需要催促或重新设定节奏
- `不愿动手` — 倾向把活儿交给 AI 或同伴

### 设计本身
- `信息层级` — 主次关系、阅读路径
- `数据准确性` — 数据源、空间精度、边界
- `视觉风格不统一` — 同一作品/系列内风格漂移
- `配色无规划` — 颜色未做整体规划
- `叙事缺失` — 缺少故事线索
- `版式松散` — 排版结构问题
- `文字处理` — 标识、注释、字体相关问题

### 协作与方向
- `跨学生协作` — 任务在团队内复用、相互借力
- `方向调整` — 项目方向变化
- `保留方案` — 重要的"为什么保留"决策
- `否定方案` — 重要的"为什么否定"决策
